Detailed Description

Redis publisher for TelemetryHub-TelemetryTaskProcessor integration.

Publishes TelemetrySample data as tasks to Redis for async processing by TelemetryTaskProcessor workers.

Integration Architecture:

TelemetryHub Device → TelemetryQueue → GatewayCore → RedisPublisher ↓ RPUSH Redis (task queue) ↓ BLPOP TelemetryTaskProcessor Workers

Usage: auto publisher = std::make_unique<RedisPublisher>("127.0.0.1", 6379); if (publisher->connect()) { TelemetrySample sample = {...}; publisher->publish_task(sample, "telemetry.analyze"); }

Publish telemetry sample as task to Redis.

Converts TelemetrySample to Task JSON format and pushes to Redis queue.

Task JSON format: { "id": "uuid-v4", "type": "telemetry.analyze", "payload": { "device_id": "sensor_001", "timestamp": "2025-12-26T10:30:00Z", "temperature": 23.5, "humidity": 65.2, "pressure": 1013.25, "voltage": 3.3, "current": 0.5 }, "priority": "NORMAL", "max_retries": 3, "created_at": "2025-12-26T10:30:00Z" }

Parameters
sampleTelemetrySample to publish
task_typeTask type (e.g., "telemetry.analyze", "telemetry.anomaly_detect")
priorityTask priority ("HIGH", "NORMAL", "LOW")
max_retriesMaximum retry attempts (default: 3)
Returns
Task ID if successful, empty string if failed
