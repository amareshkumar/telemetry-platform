\doxysubsection{Detailed Description}
RAII-\/based Redis client wrapper with connection pooling. 

Wraps redis-\/plus-\/plus library with Telemetry\+Hub-\/specific operations. Provides high-\/level interface for storing and retrieving telemetry data.

{\bfseries{Design Decisions}}\+:
\begin{DoxyItemize}
\item {\bfseries{PIMPL Pattern}}\+: Hides redis++ implementation details from clients
\item {\bfseries{Move-\/\+Only Semantics}}\+: Redis connections cannot be copied (resource ownership)
\item {\bfseries{Exception Safety}}\+: All operations provide strong exception guarantee
\item {\bfseries{Connection Pooling}}\+: Automatic pool management for concurrent access
\end{DoxyItemize}

{\bfseries{Interview Talking Points}}\+:
\begin{DoxyEnumerate}
\item {\bfseries{RAII}}\+: Why C++ destructors are critical for resource management
\item {\bfseries{PIMPL}}\+: Reduces compile-\/time dependencies, enables ABI stability
\item {\bfseries{Move Semantics}}\+: Efficient transfer of ownership (no deep copy)
\item {\bfseries{Connection Pooling}}\+: How to handle concurrent database access
\item {\bfseries{Error Handling}}\+: std\+::optional vs exceptions (when to use each)
\end{DoxyEnumerate}

{\bfseries{Thread Safety}}\+:
\begin{DoxyItemize}
\item ✅ Multiple \doxylink{classtelemetry__common_1_1_redis_client}{Redis\+Client} instances\+: Safe (separate connections)
\item ✅ Single \doxylink{classtelemetry__common_1_1_redis_client}{Redis\+Client} with connection pool\+: Safe (internally synchronized)
\item ❌ Shared \doxylink{classtelemetry__common_1_1_redis_client}{Redis\+Client} across threads\+: Unsafe without external mutex
\end{DoxyItemize}
