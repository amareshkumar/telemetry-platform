\doxysubsection{Detailed Description}
Thread-\/safe bounded priority queue for task scheduling. 

Implements a producer-\/consumer pattern with priority scheduling. Multiple producers can \doxylink{classtelemetry__processing_1_1_task_queue_ad82d3ae8f7ddc65e27ae46af9460df81}{enqueue()} tasks, multiple consumers can \doxylink{classtelemetry__processing_1_1_task_queue_ab8a1eb86b1ddcb65a2d6a2d65ee12877}{dequeue()}.

{\bfseries{Concurrency Model}}\+:
\begin{DoxyItemize}
\item Mutex protects all shared state (queue, size counters)
\item Condition variables for blocking operations\+:
\begin{DoxyItemize}
\item {\ttfamily \doxylink{classtelemetry__processing_1_1_task_queue_a0794036aed884f011ceca9fe5b5867fb}{not\+\_\+empty\+\_\+}}\+: Consumers wait when queue empty
\item {\ttfamily \doxylink{classtelemetry__processing_1_1_task_queue_a95673578555c2c90cf49178b9bf2a6eb}{not\+\_\+full\+\_\+}}\+: Producers wait when queue at capacity
\end{DoxyItemize}
\item Lock guards ensure exception-\/safe unlocking
\end{DoxyItemize}

{\bfseries{Priority Scheduling}}\+:
\begin{DoxyItemize}
\item Uses std\+::priority\+\_\+queue with custom comparator
\item Higher priority (lower enum value) processed first
\item FIFO ordering within same priority (timestamp comparison)
\end{DoxyItemize}

{\bfseries{Bounded Capacity}}\+:
\begin{DoxyItemize}
\item Prevents unbounded memory growth
\item Provides backpressure to producers
\item Configurable max size (default\+: 10,000 tasks)
\end{DoxyItemize}

{\bfseries{Interview Talking Points}}\+:
\begin{DoxyEnumerate}
\item {\bfseries{Why bounded?}} Prevent OOM in high-\/load scenarios
\item {\bfseries{Why condition variables?}} Efficient blocking (no busy-\/wait)
\item {\bfseries{Why FIFO within priority?}} Fairness + predictable latency
\item {\bfseries{Thread safety patterns}}\+: Lock guards, RAII, move semantics
\end{DoxyEnumerate}
