Thread-\/safe priority-\/based task queue for telemetry processing.

This file provides a production-\/ready priority queue implementation for high-\/throughput telemetry processing with\+:
\begin{DoxyItemize}
\item {\bfseries{Priority Scheduling}}\+: HIGH/\+MEDIUM/\+LOW priority levels
\item {\bfseries{Thread Safety}}\+: Full synchronization with mutex + condition variable
\item {\bfseries{Bounded Capacity}}\+: Configurable max size with backpressure
\item {\bfseries{Blocking Operations}}\+: enqueue() and dequeue() with timeout support
\item {\bfseries{FIFO Within Priority}}\+: Same-\/priority tasks processed in order
\end{DoxyItemize}

{\bfseries{Performance Characteristics}}\+:
\begin{DoxyItemize}
\item enqueue()\+: O(log n) -\/ binary heap insertion
\item dequeue()\+: O(log n) -\/ extract-\/min from heap
\item peek()\+: O(1) -\/ view highest priority task
\item size()/empty()\+: O(1) -\/ constant time
\end{DoxyItemize}

{\bfseries{Interview Talking Points}}\+:

\item {\bfseries{Priority Queue}}\+: Why binary heap over sorted list? (O(log n) vs O(n))
\item {\bfseries{Thread Safety}}\+: Producer-\/consumer pattern with condition variables
\item {\bfseries{Bounded Queue}}\+: Backpressure mechanism (prevent memory exhaustion)
\item {\bfseries{Timeout Handling}}\+: std\+::chrono for type-\/safe time management
\item {\bfseries{Modern C++17}}\+: std\+::optional, lambda comparators, RAII lock guards
\end{DoxyEnumerate}
